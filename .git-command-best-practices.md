# Git Command Best Practices for Scripts

**Date:** 2025-12-12  
**Issue:** Git commands without `--` separator cause ambiguous argument errors

## Rule: Always Use `--` Separator

### ❌ WRONG
```bash
git diff "$BRANCH$FILE"
```

### ✅ CORRECT
```bash
git diff "$BRANCH" -- "$FILE"
git show "$BRANCH:$FILE"  # OK: colon separates revision from path
```

## When to Use `--`

**Always use `--` when:**
- `git diff <revision> <path>` - path comes after revision
- `git log <revision> <path>` - path comes after revision
- `git checkout <revision> <path>` - path comes after revision

**No `--` needed when:**
- `git show <revision>:<path>` - colon separates revision from path

## Safe Patterns

Always quote variables:
```bash
git diff "$BRANCH" -- "$FILE"
```

## Common Mistakes

### Mistake 1: Concatenation
```bash
# ❌ WRONG
git diff "$BRANCH$FILE"

# ✅ CORRECT
git diff "$BRANCH" -- "$FILE"
```
