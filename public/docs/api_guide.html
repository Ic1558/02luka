Boss API Documentation

Overview

The Boss API is the central orchestration service for the 02luka system. It provides REST endpoints for all system operations.

Base URL

http://localhost:3000/api

Authentication

Currently uses simple bearer token authentication. Include in headers:

Authorization: Bearer YOUR_TOKEN

Endpoints

OCR Operations

POST /api/ops/ocr/approve

Approve an OCR task for execution.

Request Body:

{
  "wo_id": "WO-CLS-1234",
  "action": "publish",
  "files": [
    {
      "path": "/path/to/file.md",
      "title": "Document Title"
    }
  ]
}

Response:

{
  "status": "approved",
  "wo_id": "WO-CLS-1234",
  "approval_file": "/path/to/approval.json"
}

Risk Levels: - < 0.6 - Auto-approve - 0.6 - 0.8 - Manual review required - â‰¥ 0.8 - Auto-reject (too risky)

Calendar Operations

GET /api/calendar/events

List upcoming calendar events.

Query Parameters: - days - Number of days to look ahead (default: 30) - format - Response format: json | ics (default: json)

Response:

{
  "events": [
    {
      "id": "evt-123",
      "title": "Team Meeting",
      "start": "2025-10-31T14:00:00Z",
      "end": "2025-10-31T15:00:00Z",
      "location": "Conference Room A"
    }
  ]
}

Action Types

publish

Converts markdown to HTML and deploys to public docs site.

Handler: ~/02luka/tools/publish_doc.zsh

Files: Markdown documents, images, PDFs

Output: ~/02luka/public/docs/

ingest

Ingests documents into LightRAG knowledge base for semantic search.

Handler: ~/02luka/tools/ingest_doc.zsh (not yet implemented)

Files: Any text-based documents

archive

Long-term archival storage with compression and indexing.

Handler: ~/02luka/tools/archive_doc.zsh (not yet implemented)

Files: Any files for archival

Error Codes

Code Meaning
200 Success
400 Invalid request
401 Unauthorized
403 Forbidden (risk too high)
404 Resource not found
500 Internal server error

Rate Limits

Currently no rate limits enforced. May be added in future versions.

Telemetry

All API operations are logged to ~/02luka/g/telemetry/ in NDJSON format:

  • OCR operations: ocr_execution_YYYYMMDD.ndjson
  • Document publishing: doc_publish_YYYYMMDD.ndjson

Support

For issues or questions, check logs: - Boss API: ~/02luka/g/logs/boss_api.log - OCR Consumer: ~/02luka/g/logs/ocr_consumer.log
