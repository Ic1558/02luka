02luka System Overview

Introduction

Welcome to the 02luka system documentation. This is a test document to verify the publishing workflow.

Features

The 02luka system includes:

  • OCR Processing: Automated document approval and processing
  • Event-Driven Architecture: Inbox/processing/processed queue pattern
  • System Guards: Memory monitoring, disk swing detection, automated cleanup
  • Documentation Publishing: Markdown to HTML conversion with automatic indexing

Architecture

Core Components

  1. Boss API: Central API server handling all system operations
  2. OCR Consumer: Event-driven processor for approved documents
  3. System Guards: Proactive monitoring and maintenance
  4. Public Docs: Automated documentation publishing

Directory Structure

~/02luka/
├── bridge/
│   ├── inbox/CLC/    # Incoming approved tasks
│   ├── processing/   # Currently processing
│   ├── processed/    # Completed tasks
│   └── failed/       # Failed processing
├── f/bridge/processors/ # Consumer scripts
├── g/
│   ├── logs/         # System logs
│   ├── telemetry/    # Event tracking
│   └── locks/        # Process locks
├── public/
│   ├── docs/         # Published HTML
│   └── assets/       # CSS, images
└── tools/            # Action handlers

Usage

Publishing Documents

Documents are published through the OCR approval workflow:

  1. Create approval JSON with action: "publish"
  2. Place in ~/02luka/bridge/inbox/CLC/
  3. Consumer processes and calls publish_doc.zsh
  4. HTML output appears in ~/02luka/public/docs/

Example Approval JSON

{
  "wo_id": "WO-TEST-123",
  "action": "publish",
  "approved_by": "admin",
  "approved_at": "2025-10-31T01:30:00+0700",
  "files": [
    {
      "path": "/path/to/document.md",
      "sha256": "...",
      "title": "System Overview"
    }
  ]
}

Monitoring

System Guards

The system includes three automated guards:

  • Memory Guard (every 2 min): Alerts when RAM < 18%
  • Disk Swing Monitor (every 2 min): Detects >5GB changes in 10-minute windows
  • Auto-clean (every 48h): Removes Docker volumes and Cursor cache

Logs

  • Memory: ~/Library/Logs/02luka/memory_guard.log
  • Disk: ~/Library/Logs/02luka/disk_swing_monitor.log
  • Auto-clean: ~/Library/Logs/02luka/autoclean_48h.log
  • OCR Consumer: ~/02luka/g/logs/ocr_consumer.log
  • Publisher: ~/02luka/g/logs/publish_doc.log

Telemetry

All operations log structured events to ~/02luka/g/telemetry/:

  • ocr_execution_YYYYMMDD.ndjson - OCR consumer events
  • doc_publish_YYYYMMDD.ndjson - Publishing events

Troubleshooting

OCR Consumer Not Processing

Check the consumer log:

tail -f ~/02luka/g/logs/ocr_consumer.log

Verify LaunchAgent status:

launchctl list | grep ocr-consumer

Publishing Failures

Check publish log:

tail -f ~/02luka/g/logs/publish_doc.log

Verify pandoc is installed:

command -v pandoc
